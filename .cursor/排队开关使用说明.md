# 排队开关使用说明

## 功能说明
平台排队开关用于控制整个平台是否允许用户提交排队申请。

## 配置数据

### 添加到数据库

在 uniCloud 控制台的 `system_info` 表中添加以下数据：

```json
{
  "key": "queue_switch",
  "title": "排队开关",
  "content": "true",
  "is_published": true
}
```

### 字段说明

- **key**: `queue_switch` （固定标识，不要修改）
- **title**: `排队开关` （说明文字）
- **content**: 
  - `"true"` - 开启排队功能，用户可以提交申请
  - `"false"` - 关闭排队功能，用户无法提交申请
- **is_published**: `true` （必须设置为 true 才能生效）

## 使用方式

### 开启排队
将 `content` 字段设置为 `"true"`

```json
{
  "key": "queue_switch",
  "title": "排队开关",
  "content": "true",
  "is_published": true
}
```

### 关闭排队
将 `content` 字段设置为 `"false"`

```json
{
  "key": "queue_switch",
  "title": "排队开关",
  "content": "false",
  "is_published": true
}
```

关闭后，用户在提交排队申请时会收到提示：「当前平台暂停排队服务」

## 注意事项

1. 修改开关后立即生效，无需重启
2. 关闭排队不影响已经在排队中的任务
3. 只有管理员可以修改此配置
4. 建议在系统维护或特殊情况时临时关闭排队功能
